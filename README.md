# 🌱 My Little Community

A cozy sandbox where you create a few prompt-defined little folks — name, icon,
personality, disposition, speech style, likes/dislikes, and a set of trait
sliders — then drop them into a sunny meadow and watch them hold an autonomous
conversation. They speak one at a time in chat bubbles, develop **moods** and
**opinions of each other** that you can click to inspect live, and occasionally
get **context-aware intrusive thoughts**. You can pause, whisper world events
into the scene, and end or restart at any time.

> Built with Vite + React + TypeScript on the front end and an Express + TypeScript
> proxy on the back. The architecture deliberately separates a pure conversation
> engine from rendering and from the LLM/network layer.

---

## ✨ Features

- **Prompt-defined agents** — persona, disposition, speech style, likes/dislikes,
  five personality sliders (warmth, energy, confidence, agreeableness, emotionality),
  and talkativeness.
- **Autonomous turn-taking** — a "speaking pressure" model: loud folks dominate,
  shy folks are never starved, the same folk rarely speaks twice in a row.
- **Moods & relationships** — each turn returns structured state; click any agent
  to see their current mood and how they feel about everyone else, updating live.
- **Context-aware intrusive thoughts** — generated by a separate, configurable
  (cheaper) model based on what's actually happening, with an offline fallback.
- **Director controls** — pause, resume, end, and inject world events
  ("a bird flew across the sky…"). Collapsible story log.
- **Your API key stays server-side** — the browser only ever talks to its own origin.

## 🚀 Quick start

> Requires **Node ≥ 20** and an OpenAI API key with a small amount of credit.

```bash
# 1. clone and install (npm workspaces installs client + server)
git clone https://github.com/<you>/my-little-community.git
cd my-little-community
npm install

# 2. give the server your key
cp server/.env.example server/.env        # Windows PowerShell: Copy-Item server\.env.example server\.env
# then open server/.env and set OPENAI_API_KEY=sk-...

# 3. run client + server together
npm run dev
```

- Client (Vite): http://localhost:5173
- Server (Express API): http://localhost:8787

Vite proxies `/api/*` to the server in dev, so there's no CORS setup and the key
never reaches the browser.

### How to play

1. On the setup screen, tweak the two starter folks or add your own (2–6 total).
2. Optionally set a scene and the conversation pace.
3. Hit **Send Them Into the Meadow**.
4. Watch them talk. **Click an agent** to open their mood/opinion panel.
   Use **Pause** to whisper a world event, **End** to stop.

## ⚙️ Configuration

All configuration is environment variables in `server/.env` (see
`server/.env.example`):

| variable                  | default        | purpose                                            |
| ------------------------- | -------------- | -------------------------------------------------- |
| `OPENAI_API_KEY`          | _(required)_   | your OpenAI key — server-side only                 |
| `OPENAI_MODEL`            | `gpt-4o-mini`  | model for spoken lines + mood/opinion output       |
| `OPENAI_INTRUSIVE_MODEL`  | `gpt-4o-mini`  | smaller model for intrusive thoughts               |
| `PORT`                    | `8787`         | API port                                           |

**Model compatibility:** this app uses Chat Completions with `max_tokens`,
a custom `temperature`, and JSON mode. The **GPT-4.x family**
(`gpt-4o-mini`, `gpt-4.1`, `gpt-4.1-mini`, `gpt-4.1-nano`) works out of the box.
Newer reasoning models (GPT-5 / o-series) reject `max_tokens`/`temperature` and
require `max_completion_tokens`, so they need a small code change before use.

## 🗂️ Architecture

```
my-little-community/
├── server/   # stateless proxy: holds the key, builds prompts, calls OpenAI
│   └── src/  # env · schema (zod contract) · llm/ (provider seam) ·
│             # conversation/ (pure prompts) · routes/ (generate, intrusive)
└── client/   # Vite + React + TS, DOM/SVG rendering
    └── src/  # domain/ · engine/ (pure TurnEngine + ConversationEngine) ·
               # api/ · state/ (zustand) · components/
```

- The **conversation loop** lives on the client; the **server is a stateless
  proxy**. The client calls `POST /api/generate` for one turn at a time.
- **zod is the single source of truth** at the API boundary — it validates
  requests and generates the types.
- **`ConversationEngine` knows nothing about React/DOM/network** — it receives
  injected ports, which is what makes it unit-testable.
